Fondeos · plata que cargás a las cuentas

Cada vez que cargás plata a una cuenta de un cliente. Sirve para saber cuánto le tirás por día. Si es con Pillín, pagás la mitad. La ganancia no se cuenta acá (se cuenta en la Diaria sobre lo gastado) — esto es solo el control de lo cargado.

Diaria · cierres diarios

El cierre diario de cada cliente: ponés los mensajes metidos y las cargas, y la app te dice el gasto (mensajes × costo del publicista) o lo ponés manual (para vos y Santi, que el costo varía). Te muestra el presupuesto sobrante, la conversión y el rendimiento. La ganancia Pillín ahora se registra en la solapa Fondeos.

Pillín · cargas al 50%

Lo que vos y Santi manejan con el sistema de duplicación. Por cada fondeo marcado "Pillín", el 50% de lo cargado es ganancia inmediata (pagás la mitad). Acá lo ves por cliente, unificado en USDT al TC de Config.

Conectar la nube

Para tener login y datos compartidos con Tomi en tiempo real, la app usa Supabase (gratis). Pegá los datos de tu proyecto. Es una sola vez.

📖 ¿Cómo lo consigo? (paso a paso, ~5 min)

1. Entrá a supabase.com, creá cuenta gratis y hacé New project (elegí una contraseña y esperá 1-2 min a que se cree).

2. En el menú Project Settings → API, copiá el Project URL y la anon public key.

3. Andá a SQL Editor, pegá esto y apretá Run:

create table if not exists app_state ( id text primary key, data jsonb, updated_at timestamptz default now() ); alter table app_state enable row level security; create policy "auth_read" on app_state for select to authenticated using (true); create policy "auth_insert" on app_state for insert to authenticated with check (true); create policy "auth_update" on app_state for update to authenticated using (true);

4. En Authentication → Providers → Email, desactivá "Confirm email" (así entran sin verificar el mail).

5. Volvé acá, pegá el URL + la key y tocá Conectar. Después creás tu usuario y el de Tomi.
